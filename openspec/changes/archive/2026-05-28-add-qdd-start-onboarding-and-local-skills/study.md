## Question

How should QDD add a project-onboarding workflow that captures project resources and local skill authority before the first study begins?

## Hypothesis / Expectation

If QDD adds a fifth bootstrap surface named `qdd-start`, extends the scaffold with `data/` and `.agents/skills/`, and exposes a minimal project-level instructions target, then project onboarding will become repeatable without turning QDD into a project manager or a heavier orchestration system.

## Inputs

- current scaffold creation in `src/commands/init.ts`
- current bootstrap projection in `src/runtime/bootstrap.ts`
- current project defaults in `src/runtime/defaults.ts`
- current path contract in `src/runtime/constants.ts`
- current instructions builder in `src/runtime/instructions.ts`
- current validation and inspection behavior in `src/runtime/inspection.ts`
- current docs that already describe `context/resources.md` as the project-level resource entrypoint
- user requirement that data should be linked into the project through symlinks under `data/`
- user requirement that the agent should only rely on project-owned skills, using a local `.agents/skills/`-style pattern

## Evidence Plan

- a documented `qdd-start` onboarding surface that is generated by `qdd init`
- a scaffold that includes `data/` and `.agents/skills/`
- a project-level instructions target for onboarding work
- a local-skill resolution rule for later study/task instructions
- a readable convention for writing research theme, biological background, environment, and datasets into `contract.yaml` and `context/resources.md`
- a symlink-based dataset import convention under `data/`
- validation coverage for broken dataset links, placeholder onboarding state, and missing local skills

## Blockers

- tool runtimes may still expose global skills, so local-skill authority can only be enforced at the QDD protocol and prompt layer in this slice
- dataset linking behavior must stay simple and cross-platform enough for the current Node CLI
- onboarding must stay lightweight and should not become an interactive AI-calling CLI
- this slice should not quietly grow into a full project-layer redesign

## Exit Signal

This study is ready to close when QDD has a concrete onboarding contract that keeps `qdd init` lightweight, gives agents an explicit place to write project context, introduces a trustworthy local skill boundary, and keeps the later study loop unchanged apart from consuming that new project context.
